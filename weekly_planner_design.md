# 週案エディタ｜第1層（週案コア）詳細設計書

## 0. このドキュメントの位置づけ

本書は週案エディタの**第1層（週案コア）**の設計書である。参考実装は `weekly_planner_core.html` に置く。受領後はCodexで継続開発し、第2層（時数エンジン）・第3層（外部連携）を育てていく。

設計の最優先事項：**ブラウザ内完結・データ流出なし・教員がインストール不要で使える**。

---

## 1. 第1層のスコープ

| 含む（第1層で実装） | 含まない（第2層以降） |
|---|---|
| 時間割グリッド（行事/業前/1〜6限） | 時数自動計算・進捗割り出し |
| 月〜金が1画面に収まる表示 | 教科書年間指導計画の読み込み |
| 各コマのメモ（単元・小単元・持ち物） | Googleカレンダー連携 |
| Excel年間行事の一括読み込み | （Googleタスク連携は見送り、アプリ内タスクで代替） |
| 週送り（前週コピー→微修正） | |
| アプリ内タスク欄（今日のタスク） | |
| AIプロンプト生成＆コピー | |
| A4横印刷 | |
| localStorage保存 | |

第1層を「確実に動く土台」として完成させることを目標とする。第2層以降の機能は、**データ構造に拡張ポイントを用意**しておくことで後付けを容易にする。

### データ構造の設計意図（Codexへの申し送り）

- **`meta.grade`** は第2層の時数計算で必須。第1層から持たせておく。
- **`subjects[].id`** で全コマが教科を参照する。教科の追加・色変更があってもコマ側は壊れない。
- **`weeks` は週ごとに独立**。週送り（前週コピー）は週オブジェクトをディープコピーして新しい `weekStart` キーで保存するだけ。
- **`periods` のキーは文字列の `"1"`〜`"6"`**。行事欄・業前欄は `periods` の外に置くことで、限のループと分離する。
- **`events` は日付キーのマップ**。Excel読み込みはこのマップを生成するだけ。第2層の「祝日で時数が…」の警告もここを参照する。
- 第2層の時数計算は **`weeks` 全体を走査して `subjectId` をカウント**すれば実装できる。

### 3-2. グリッドの行構成

| 行 | 高さ | 内容 |
|---|---|---|
| ヘッダー行 | 小 | 曜日＋日付（月13 / 火14…） |
| **行事行** | **小（通常コマの50%）** | 学校行事（Excel読み込み or 手動） |
| **業前行** | **小（通常コマの50%）** | 朝読書・朝計算など自由テキスト |
| 1〜6限行 | 標準 | 教科コマ（D&D配置＋メモ） |

**月〜金1画面化のキモ**：CSS Grid で `grid-template-rows` を「ヘッダー auto / 行事 0.5fr / 業前 0.5fr / 1限〜6限 各1fr」とし、全体を `height:100vh` 内に収める。

### 3-3. コマの表示と編集

- **空きコマ**：薄いグレーで「＋」。クリックまたは教科パレットからD&Dで教科を配置。
- **配置済みコマ**：教科色の背景＋教科名＋単元名（小さく）。クリックで編集ポップアップ。
- **編集ポップアップ**：教科選択／単元名／小単元名／持ち物／メモ／AIプロンプト生成ボタン／削除。

## 4. 各機能の仕様

### 4-1. 教科パレット（サイドパネル）

- 教科チップを縦に並べる。各チップは教科色。
- チップをグリッドのコマにドラッグ&ドロップで配置。
- HTML5 Drag and Drop API を使用（`draggable`, `onDragStart`, `onDrop`）。
- 教科の追加・色変更・並べ替えは設定画面で行う。第1層では最低限、追加と色変更を実装する。

### 4-2. Excel年間行事の読み込み

- SheetJS（xlsx）をCDN読み込み。
- ファイル選択 → シートを読む → **列マッピングUI**を表示。
- 学校ごとにExcelフォーマットが違うため、固定パースは不可。マッピング必須。
- 日付の解釈：Excelシリアル値・文字列日付の両対応。
- 読み込み結果を `events` マップにマージ。プレビュー後に確定。
- 取り込み前に既存 `events` を上書きするか追加するか選ばせる。

### 4-3. 週送り

- ヘッダーの [◀前週] [次週▶] で `weekStart` を±7日。
- その週のデータが無ければ空グリッドを生成。
- **[前週をコピー]ボタン**：直前の週の `days` 構造をディープコピーして現在週に複製する。

### 4-4. アプリ内タスク欄

- サイドパネルに「今日のタスク」リスト。
- テキスト入力＋日付（デフォルト今日）でタスク追加。
- チェックで完了（取り消し線）、削除可能。
- 今日の日付のタスクを上部に表示。
- `tasks` 配列に保存。**第3層でGAS×Googleタスク連携する場合に備え、タスクの読み書きを関数（`loadTasks`/`saveTasks`）に分離**しておく。

### 4-5. AIプロンプト生成

- コマ編集ポップアップの「AIプロンプト生成」ボタン。
- 選択コマの教科・単元・小単元から定型プロンプトを生成してクリップボードにコピー。
- プロンプトの雛形は「授業展開」「板書計画」「ワークシート案」「評価規準」を用意する。

### 4-6. 印刷

- `@media print` でサイドパネル・ヘッダーを非表示。
- A4横で週案簿の体裁。
- `@page { size: A4 landscape; margin: 8mm; }`

## 5. localStorage 保存フロー

```javascript
function savePlan(state) {
  try {
    localStorage.setItem('weeklyPlan_v1', JSON.stringify(state));
  } catch(e) {
    console.warn('localStorage unavailable', e);
  }
}

function loadPlan() {
  try {
    const raw = localStorage.getItem('weeklyPlan_v1');
    return raw ? JSON.parse(raw) : createInitialState();
  } catch(e) {
    return createInitialState();
  }
}

function createInitialState() {
  return {
    version: 1,
    meta: { grade: 5, className: '1', teacherName: '', schoolYear: 2026 },
    subjects: DEFAULT_SUBJECTS,
    events: {},
    weeks: {},
    tasks: []
  };
}
```

Codexへの申し送り：参考実装はReact stateで動く形にしてある。`savePlan` / `loadPlan` は分離済みなので、本番では `useEffect` で `savePlan` をstate変更時に呼ぶだけでlocalStorage保存が有効になる。

## 7. 第2層実装済み範囲：時数カウンター

今回 `weekly_planner_core.html` に実装する第2層の範囲は、**時数カウンター**のみとする。

### 7-1. データ入力

- `meta.grade`
- `subjects`
- `weeks`

### 7-2. 標準時数データ

HTML内に `STANDARD_HOURS` 定数を持つ。学校や年度の事情で調整できるよう、実装コメントに「初期値・要確認」と明示する。

```javascript
const STANDARD_HOURS = {
  5: {
    kokugo: 175,
    shakai: 100,
    sansu: 175,
    rika: 105,
    ongaku: 50,
    zukou: 50,
    katei: 60,
    taiiku: 90,
    doutoku: 35,
    gaikoku: 70,
    sougou: 70,
    tokkatsu: 35
  }
};
```

### 7-3. 集計ルール

- `weeks` 全体を走査する。
- 各 `day.periods["1"]`〜`["6"]` のうち、`slot && slot.subjectId` があるものを1コマとして数える。
- 行事行、業前欄、空きコマは数えない。
- 現時点では「計画済み時数」として扱い、実施済み・未実施の区別はしない。

### 7-4. UI

サイドパネルに「時数カウンター」を追加する。

表示項目:

- 教科名
- 計画時数
- 標準時数
- 残り時数
- 進捗バー

標準時数が未設定の教科は「標準未設定」と表示し、カウントだけ表示する。

## 9. 第1層の完成判定

以下がすべて動けば第1層完成とする。

1. 月〜金 × (行事/業前/1〜6限) のグリッドが1画面に収まる
2. 教科パレットからD&Dでコマに教科を配置できる
3. コマをクリックして単元・小単元・持ち物・メモを編集できる
4. 前週/次週に移動でき、前週コピーができる
5. Excel年間行事を列マッピングで読み込み、行事行に表示される
6. アプリ内タスクの追加・完了・削除ができる
7. コマからAIプロンプトを生成してコピーできる
8. A4横で印刷できる
9. localStorageに保存され、再読み込みで復元される

第2層の今回完成判定:

1. `meta.grade` に応じて標準時数を参照できる
2. `weeks` 全体から教科ごとの計画時数を集計できる
3. 標準時数との差分と進捗率を表示できる
